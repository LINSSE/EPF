Activity: Especificación de Requisitos del Software
Esta actividad tiene como objeto establecer un procedimiento general para la gestión de requisitos del software, que sirva para gestionar de manera sistemática todos los procesos relativos al correcto tratamiento de los requerimientos de software dentro del proyecto, cumpliendo estos con las pautas de seguridad establecidas en el mismo.
DescriptionWork Breakdown StructureTeam AllocationWork Product Usage
Purpose
Este procedimiento general se usa para gestionar los requisitos del software de aplicaciones ferroviarias de acuerdo con la normativa UNE-EN 50128.
Relationships
Parent Activities
  • Gestión de los Requisitos del Software
Description

  • Las tareas de: obtención, especificación y verificación relacionados a la Gestión de los Requerimientos del Software y sus secuencias se ilustran en el workflow.

  • Durante la tarea, cuando se utilicen escalas de valoración, las mismas irán del 1 al 10, siendo 1 el valor menos y 10 el más significativo.
